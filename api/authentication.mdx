The Punto Diez API uses API keys to authenticate every request. You generate keys inside your dashboard and pass them in the `Authorization` header of each call. Keys are scoped, so you can grant only the permissions each integration actually needs.

## Generating an API Key

<Steps>
  <Step title="Log in to Punto Diez">
    Open your Punto Diez dashboard and sign in to the workspace where you want to create the key.
  </Step>
  <Step title="Open API Keys settings">
    Navigate to **Settings → API Keys** in the left sidebar.
  </Step>
  <Step title="Create a new key">
    Click **New API Key**. Enter a descriptive label — for example, `Production Integration` or `Zapier Connector` — so you can identify the key later.
  </Step>
  <Step title="Select scopes">
    Choose the scopes this key needs. Grant only the permissions required for your use case.
  </Step>
  <Step title="Copy the key">
    Click **Create**. Your new API key will appear on screen. Copy it immediately and store it somewhere secure.
  </Step>
</Steps>

<Warning>
  Your API key is displayed **only once** at the moment of creation. If you navigate away without copying it, you will need to delete the key and create a new one. Never commit API keys to source control.
</Warning>

## Using Your API Key

Pass your API key in the `Authorization` header of every request using the `Bearer` scheme:

<CodeGroup>
```bash curl
curl -H "Authorization: Bearer YOUR_API_KEY" \
  https://api.puntodiez.com/v1/projects
```

```javascript JavaScript
const response = await fetch('https://api.puntodiez.com/v1/projects', {
  method: 'GET',
  headers: {
    'Authorization': 'Bearer YOUR_API_KEY',
    'Content-Type': 'application/json',
  },
});

const { data } = await response.json();
console.log(data);
```
</CodeGroup>

Replace `YOUR_API_KEY` with the actual key you copied from the dashboard. Never expose your key in client-side code or public repositories.

## API Key Scopes

Each key can be assigned one or more scopes that control what the key can access. Requests made with a key that lacks the required scope receive a `403 Forbidden` response.

| Scope | What it allows |
|---|---|
| `read:projects` | Read project data, list projects, and retrieve project details. |
| `write:projects` | Create, update, and delete projects. |
| `read:users` | Read user profiles and list workspace members. |
| `write:users` | Update user data and roles. |
| `webhooks` | Create, list, and delete webhooks. |

Assign the minimum scopes necessary for your integration to follow the principle of least privilege.

## Authentication Errors

When authentication fails, the API returns one of the following HTTP errors:

| Status Code | Meaning |
|---|---|
| `401 Unauthorized` | The `Authorization` header is missing, malformed, or contains an invalid or revoked API key. |
| `403 Forbidden` | The API key is valid, but it does not have the scope required to perform the requested action. |

A `401` response looks like this:

<CodeGroup>
```bash curl
curl -i -H "Authorization: Bearer INVALID_KEY" \
  https://api.puntodiez.com/v1/projects
```

```javascript JavaScript
const response = await fetch('https://api.puntodiez.com/v1/projects', {
  headers: {
    'Authorization': 'Bearer INVALID_KEY',
  },
});

if (response.status === 401) {
  const { error } = await response.json();
  console.error(error.message); // "Invalid or missing API key."
}
```
</CodeGroup>

```json Response
{
  "error": {
    "code": "unauthorized",
    "message": "Invalid or missing API key."
  }
}
```

A `403` response means you need to regenerate your key with the required scope or contact your workspace owner to update permissions.
