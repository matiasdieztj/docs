The Projects API lets you create, retrieve, update, and delete projects inside any workspace your API key has access to. Use it to automate project setup, sync project data to external systems, or build custom dashboards that surface real-time project status.

---

## GET /projects

Retrieve a paginated list of all projects in a workspace.

<CodeGroup>
```bash curl
curl -G https://api.puntodiez.com/v1/projects \
  -H "Authorization: Bearer YOUR_API_KEY" \
  -d workspace_id=ws_123 \
  -d page=1 \
  -d per_page=20 \
  -d status=active
```

```javascript JavaScript
const params = new URLSearchParams({
  workspace_id: 'ws_123',
  page: '1',
  per_page: '20',
  status: 'active',
});

const response = await fetch(`https://api.puntodiez.com/v1/projects?${params}`, {
  headers: {
    'Authorization': 'Bearer YOUR_API_KEY',
  },
});

const { data, meta } = await response.json();
console.log(data);
```
</CodeGroup>

### Query Parameters

<ParamField query="workspace_id" type="string" required>
  The ID of the workspace whose projects you want to list.
</ParamField>

<ParamField query="page" type="integer">
  Page number to retrieve. Defaults to `1`.
</ParamField>

<ParamField query="per_page" type="integer">
  Number of projects per page. Defaults to `20`, maximum `100`.
</ParamField>

<ParamField query="status" type="string">
  Filter projects by status. Accepted values: `active`, `archived`. Omit to return all projects.
</ParamField>


<ResponseField name="data" type="array">
  Array of project objects.

  <Expandable title="Project object fields">

## POST /projects

Create a new project inside a workspace.

<CodeGroup>
```bash curl
curl -X POST https://api.puntodiez.com/v1/projects \
  -H "Authorization: Bearer YOUR_API_KEY" \
  -H "Content-Type: application/json" \
  -d '{
    "workspace_id": "ws_123",
    "name": "Launch Campaign",
    "description": "Q3 go-to-market campaign.",
    "due_date": "2024-09-30",
    "view": "board"
  }'
```

```javascript JavaScript
const response = await fetch('https://api.puntodiez.com/v1/projects', {
  method: 'POST',
  headers: {
    'Authorization': 'Bearer YOUR_API_KEY',
    'Content-Type': 'application/json',
  },
  body: JSON.stringify({
    workspace_id: 'ws_123',
    name: 'Launch Campaign',
    description: 'Q3 go-to-market campaign.',
    due_date: '2024-09-30',
    view: 'board',
  }),
});

const { data } = await response.json();
console.log(data);
```
</CodeGroup>

### Body Parameters

<ParamField body="workspace_id" type="string" required>
  The ID of the workspace in which to create the project.
</ParamField>

<ParamField body="name" type="string" required>
  The display name for the new project. Must be between 1 and 255 characters.
</ParamField>

<ParamField body="description" type="string">
  Optional description of the project's purpose or goals.
</ParamField>

<ParamField body="due_date" type="string">
  Target completion date in `YYYY-MM-DD` format.
</ParamField>

<ParamField body="view" type="string">
  Default view for the project. Accepted values: `list`, `board`, `timeline`. Defaults to `list`.
</ParamField>

### Response

Returns the newly created project object with HTTP `201 Created`.

```json
{
  "data": {
    "id": "proj_abc123",
    "name": "Launch Campaign",
    "description": "Q3 go-to-market campaign.",
    "status": "active",
    "due_date": "2024-09-30",
    "view": "board",
    "created_at": "2024-06-10T12:00:00Z",
    "updated_at": "2024-06-10T12:00:00Z"
  }
}
```

<ResponseField name="data" type="object">
  The newly created project object.

  <Expandable title="Created project object fields">
    <ResponseField name="id" type="string">
      Unique identifier for the project.
    </ResponseField>
    <ResponseField name="name" type="string">
      Display name of the project.
    </ResponseField>
    <ResponseField name="description" type="string">
      Optional description of the project's purpose or goals.
    </ResponseField>
    <ResponseField name="status" type="string">
      Initial status of the project. Always `active` on creation.
    </ResponseField>
    <ResponseField name="due_date" type="string">
      Target completion date in `YYYY-MM-DD` format, if provided.
    </ResponseField>
    <ResponseField name="view" type="string">
      Default view for the project. One of `list`, `board`, or `timeline`.
    </ResponseField>
    <ResponseField name="created_at" type="string">
      ISO 8601 timestamp for when the project was created.
    </ResponseField>
    <ResponseField name="updated_at" type="string">
      ISO 8601 timestamp for when the project was last updated.
    </ResponseField>
  </Expandable>
</ResponseField>

## GET /projects/{id}

Retrieve a single project by its ID, including membership details, task counts, and progress.

<CodeGroup>
```bash curl
curl https://api.puntodiez.com/v1/projects/proj_abc123 \
  -H "Authorization: Bearer YOUR_API_KEY"
```

```javascript JavaScript
const response = await fetch('https://api.puntodiez.com/v1/projects/proj_abc123', {
  headers: {
    'Authorization': 'Bearer YOUR_API_KEY',
  },
});

const { data } = await response.json();
console.log(data);
```
</CodeGroup>

### Path Parameters

<ParamField path="id" type="string" required>
  The unique ID of the project to retrieve.
</ParamField>

### Response

```json
{
  "data": {
    "id": "proj_abc123",
    "name": "Launch Campaign",
    "description": "Q3 go-to-market campaign.",
    "status": "active",
    "due_date": "2024-09-30",
    "view": "board",
    "task_count": 34,
    "progress_percent": 62,
    "members": [
      {
        "id": "usr_001",
        "name": "Ana Torres",
        "role": "owner",
        "avatar_url": "https://cdn.puntodiez.com/avatars/usr_001.png"
      },
      {
        "id": "usr_002",
        "name": "Marco Reyes",
        "role": "member",
        "avatar_url": "https://cdn.puntodiez.com/avatars/usr_002.png"
      }
    ],
    "created_at": "2024-06-10T12:00:00Z",
    "updated_at": "2024-06-18T09:15:00Z"
  }
}
```

<ResponseField name="data" type="object">
  Full project object.

  <Expandable title="Additional fields on single-project responses">
    <ResponseField name="task_count" type="integer">
      Total number of tasks in the project, across all statuses.
    </ResponseField>
    <ResponseField name="progress_percent" type="integer">
      Percentage of completed tasks out of the total task count, from `0` to `100`.
    </ResponseField>
    <ResponseField name="members" type="array">
      Array of member objects, each containing `id`, `name`, `role`, and `avatar_url`.
    </ResponseField>
  </Expandable>
</ResponseField>

## PUT /projects/{id}

Update one or more fields on an existing project. Send only the fields you want to change.

<CodeGroup>
```bash curl
curl -X PUT https://api.puntodiez.com/v1/projects/proj_abc123 \
  -H "Authorization: Bearer YOUR_API_KEY" \
  -H "Content-Type: application/json" \
  -d '{
    "name": "Q3 Launch Campaign",
    "status": "active",
    "due_date": "2024-10-15",
    "view": "timeline"
  }'
```

```javascript JavaScript
const response = await fetch('https://api.puntodiez.com/v1/projects/proj_abc123', {
  method: 'PUT',
  headers: {
    'Authorization': 'Bearer YOUR_API_KEY',
    'Content-Type': 'application/json',
  },
  body: JSON.stringify({
    name: 'Q3 Launch Campaign',
    status: 'active',
    due_date: '2024-10-15',
    view: 'timeline',
  }),
});

const { data } = await response.json();
console.log(data);
```
</CodeGroup>

### Path Parameters

<ParamField path="id" type="string" required>
  The unique ID of the project to update.
</ParamField>

### Body Parameters

<ParamField body="name" type="string">
  New display name for the project.
</ParamField>

<ParamField body="description" type="string">
  Updated project description.
</ParamField>

<ParamField body="status" type="string">
  New status for the project. Accepted values: `active`, `archived`.
</ParamField>

<ParamField body="due_date" type="string">
  Updated target completion date in `YYYY-MM-DD` format.
</ParamField>

<ParamField body="view" type="string">
  Updated default view. Accepted values: `list`, `board`, `timeline`.
</ParamField>

The response returns the full updated project object with HTTP `200 OK`.

```json
{
  "data": {

```

<ResponseField name="data" type="object">

## DELETE /projects/{id}

Permanently delete a project and all of its associated tasks, comments, and attachments.

<Warning>
  This action is **irreversible**. Deleting a project immediately and permanently removes all tasks, comments, files, and activity history associated with it. There is no way to recover a deleted project.
</Warning>

<CodeGroup>
```bash curl
curl -X DELETE https://api.puntodiez.com/v1/projects/proj_abc123 \
  -H "Authorization: Bearer YOUR_API_KEY"
```

```javascript JavaScript
const response = await fetch('https://api.puntodiez.com/v1/projects/proj_abc123', {
  method: 'DELETE',
  headers: {
    'Authorization': 'Bearer YOUR_API_KEY',
  },
});

if (response.status === 204) {
  console.log('Project deleted successfully.');
}
```
</CodeGroup>

### Path Parameters

<ParamField path="id" type="string" required>
  The unique ID of the project to delete.
</ParamField>

A successful deletion returns HTTP `204 No Content` with an empty response body.
